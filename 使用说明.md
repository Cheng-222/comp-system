# 竞赛管理系统使用说明

## 1. 项目介绍

竞赛管理系统是一个面向校内外竞赛组织与管理场景的综合平台，围绕赛事配置、学生报名、材料提交、资格审核、状态流转、消息通知与赛后归档构建了完整的闭环管理流程。

### 1.1 系统架构

- **前端**：Vue 3 + Vite + Element Plus
- **后端**：Flask + SQLAlchemy
- **数据库**：MySQL 8.0（支持通过 DATABASE_URL 切换到 SQLite）
- **部署**：Docker Compose

### 1.2 主要功能

- **认证与权限**：登录、当前用户、菜单加载、按角色控制可见范围与操作权限
- **系统基础**：工作台统计、字典数据
- **学生管理**：列表、详情、新增、编辑、启停，展示最近报名与赛后结果
- **赛事管理**：列表、详情、新增、编辑、发布、状态流转、负责人/审核人分配
- **报名管理**：列表、详情、创建、编辑、批量导入、导出、材料提交、材料下载、审核、补正、退赛、换人、补录、流转日志
- **材料与资格**：审核队列处理、通过/补正/驳回、资格状态调整、有效名单与历史变更查看
- **通知中心**：消息列表、创建、编辑、发送、取消、已读，支持按赛事、角色、状态定向触达
- **赛后管理**：成绩列表、详情、新增、编辑、批量导入、证书上传、证书下载、归档导出
- **统计报表**：统计概览、获奖分析、未获奖名单、报表导出
- **附件能力**：报名材料真实上传下载、成绩导入/报名导入/证书上传统一支持拖拽或点击上传

## 2. 环境准备

### 2.1 前端环境

- **Node.js**：推荐 v16.0 及以上
- **npm**：推荐 v8.0 及以上

### 2.2 后端环境

- **Python**：推荐 v3.7 及以上
- **MySQL**：8.0 版本（或使用 SQLite 进行开发验证）

### 2.3 其他工具

- **Git**：用于代码版本控制
- **Docker**（可选）：用于容器化部署

## 3. 安装与启动

### 3.1 代码获取

```bash
git clone https://gitee.com/wang-haotian0877/comp-manage.git
cd comp-manage
```

### 3.2 后端启动

1. 安装依赖

```bash
python3 -m pip install --target backend/.deps -r backend/requirements.txt
```

2. 启动后端服务

```bash
PYTHONPATH=backend/.deps:backend python3 backend/run.py
```

默认启动在 `http://localhost:5000`，可通过环境变量 `APP_PORT` 修改端口。

### 3.3 前端启动

1. 安装依赖

```bash
cd frontend
npm install
```

2. 启动前端开发服务

```bash
npm run dev
```

默认启动在 `http://localhost:5173`。

### 3.4 Docker Compose 启动（推荐）

如果已安装 Docker，可使用 Docker Compose 一键启动整个系统：

```bash
docker compose up
```

系统会自动构建并启动前端、后端和数据库服务。

### 3.5 环境配置

- 前端配置：如需修改后端API地址，可编辑 `frontend/.env.development` 文件中的 `VITE_PROXY_TARGET` 配置
- 后端配置：可通过 `.env` 文件或环境变量修改数据库连接等配置

## 5. 功能使用指南

### 5.1 工作台

登录系统后，首先进入工作台页面，展示系统概览和统计信息，包括：
- 赛事数量统计
- 报名人数统计
- 审核状态统计
- 最近活动记录

### 5.2 学生管理

**功能说明**：管理系统中的学生用户，包括新增、编辑、查看详情等操作。

**操作步骤**：
1. 点击左侧菜单「学生管理」
2. 查看学生列表，支持分页和搜索
3. 点击「新增」按钮添加新学生
4. 点击「编辑」按钮修改学生信息
5. 点击「详情」按钮查看学生详细信息，包括报名记录和比赛成绩

**技术实现**：
- 前端：使用 Element Plus 的表格组件展示学生列表，表单组件用于新增和编辑
- 后端：通过 Flask 蓝图 `students.py` 处理学生相关的 API 请求
- 数据库：使用 SQLAlchemy ORM 操作学生表

### 5.3 赛事管理

**功能说明**：管理竞赛赛事，包括创建、编辑、发布、状态管理等。

**操作步骤**：
1. 点击左侧菜单「赛事管理」
2. 查看赛事列表，支持按状态筛选
3. 点击「新增」按钮创建新赛事
4. 填写赛事基本信息、报名时间、比赛时间等
5. 分配赛事负责人和审核人员
6. 发布赛事，使其对学生可见

**技术实现**：
- 前端：使用 Vue 组件构建赛事表单，支持富文本编辑
- 后端：通过 `contests.py` 蓝图处理赛事相关请求
- 状态管理：使用工作流状态机管理赛事状态流转

### 5.4 报名管理

**功能说明**：管理学生报名信息，包括创建报名、材料提交、审核等。

**操作步骤**：
1. 点击左侧菜单「报名管理」
2. 查看报名列表，支持按赛事和状态筛选
3. 点击「新增」按钮创建新报名
4. 批量导入学生报名信息（支持Excel导入）
5. 查看报名详情，上传报名材料
6. 审核报名信息，可通过、驳回或要求补正

**技术实现**：
- 前端：文件上传组件支持拖拽上传，表格组件展示报名列表
- 后端：`registrations.py` 蓝图处理报名相关请求
- 材料管理：文件系统存储上传的材料，数据库记录文件路径

### 5.5 材料与资格审核

**功能说明**：集中处理需要审核的材料和资格申请。

**操作步骤**：
1. 点击左侧菜单「资格审核」
2. 查看审核队列，按优先级排序
3. 点击「审核」按钮处理审核项
4. 查看材料详情，进行审核操作
5. 记录审核意见，更新资格状态

**技术实现**：
- 前端：审核工作流界面，支持批量操作
- 后端：审核逻辑处理，状态更新
- 日志记录：记录审核操作和状态变更历史

### 5.6 通知中心

**功能说明**：发送系统通知，支持定向触达不同角色的用户。

**操作步骤**：
1. 点击左侧菜单「通知中心」
2. 查看通知列表，包括已发送和未发送的通知
3. 点击「新增」按钮创建新通知
4. 设置通知标题、内容、发送范围（赛事、角色、状态）
5. 发送通知或保存为草稿

**技术实现**：
- 前端：通知编辑界面，支持富文本内容
- 后端：`messages.py` 蓝图处理通知相关请求
- 消息推送：根据定向规则发送通知到目标用户

### 5.7 赛后管理

**功能说明**：管理比赛成绩和证书，包括成绩录入、证书上传等。

**操作步骤**：
1. 点击左侧菜单「赛后管理」
2. 查看成绩列表，支持按赛事筛选
3. 点击「新增」按钮录入成绩
4. 批量导入成绩（支持Excel导入）
5. 上传证书，支持批量上传
6. 导出成绩和证书

**技术实现**：
- 前端：成绩录入表单，证书上传组件
- 后端：`results.py` 蓝图处理成绩相关请求
- 证书管理：文件系统存储证书文件

### 5.8 统计报表

**功能说明**：生成各类统计报表，包括赛事统计、获奖分析等。

**操作步骤**：
1. 点击左侧菜单「统计报表」
2. 查看统计概览，包括参赛人数、获奖情况等
3. 查看获奖分析，按奖项等级和赛事分类
4. 查看未获奖名单
5. 导出报表为Excel格式

**技术实现**：
- 前端：使用 ECharts 生成图表，表格展示详细数据
- 后端：统计查询逻辑，数据聚合处理
- 导出功能：使用 Excel 工具生成报表文件

## 6. 技术实现说明

### 6.1 前端技术

**Vue 3 + Vite**：
- 使用组合式 API 构建组件
- Vite 作为构建工具，提供快速的开发体验
- 路由管理使用 Vue Router 4
- 状态管理使用 Pinia

**Element Plus**：
- 提供丰富的 UI 组件，如表格、表单、对话框等
- 支持主题定制和国际化
- 响应式设计，适配不同设备

**其他前端库**：
- Axios：处理 HTTP 请求
- ECharts：数据可视化
- Vue Quill：富文本编辑器
- FileSaver：文件下载

### 6.2 后端技术

**Flask**：
- 轻量级 Web 框架，提供路由和视图功能
- 蓝图系统，模块化组织代码
- 中间件机制，处理请求和响应

**SQLAlchemy**：
- ORM 框架，简化数据库操作
- 支持多种数据库后端
- 提供事务和会话管理

**其他后端库**：
- Flask-CORS：处理跨域请求
- PyJWT：JSON Web Token 认证
- Excel 处理库：支持 Excel 文件导入导出

### 6.3 数据库设计

系统使用关系型数据库，主要表结构包括：
- 用户表：存储用户信息和权限
- 赛事表：存储赛事基本信息
- 报名表：存储学生报名信息
- 材料表：存储上传的材料信息
- 成绩表：存储比赛成绩
- 通知表：存储系统通知
- 日志表：记录系统操作日志

### 6.4 权限控制

系统采用基于角色的权限控制（RBAC）：
- 管理员：拥有所有权限
- 竞赛教师：管理赛事和报名
- 审核人员：处理材料和资格审核
- 学生：报名参赛和查看个人信息

权限控制通过前端路由守卫和后端 API 权限检查实现。

### 6.5 文件管理

系统使用本地文件系统存储上传的文件：
- 上传目录：`uploads/`
- 文件路径存储在数据库中
- 支持文件上传、下载和预览

## 7. 常见问题与解决方案

### 7.1 启动问题

**问题**：后端启动失败，提示数据库连接错误
**解决方案**：
- 检查数据库服务是否运行
- 检查数据库连接配置是否正确
- 尝试使用 SQLite 进行开发验证（修改 DATABASE_URL）

**问题**：前端启动后无法连接后端
**解决方案**：
- 检查后端服务是否正常运行
- 检查 `frontend/.env.development` 中的代理配置
- 确保前端和后端端口没有被占用

### 7.2 功能问题

**问题**：无法上传文件
**解决方案**：
- 检查 `uploads/` 目录是否存在且有写入权限
- 检查文件大小是否超过限制
- 检查文件类型是否被允许

**问题**：批量导入失败
**解决方案**：
- 检查 Excel 文件格式是否正确
- 检查数据是否符合要求
- 查看系统日志获取详细错误信息

### 7.3 性能问题

**问题**：系统响应缓慢
**解决方案**：
- 检查数据库索引是否正确创建
- 优化查询语句，避免全表扫描
- 考虑使用缓存减少数据库查询
