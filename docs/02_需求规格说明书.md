# 需求规格说明书

## 1. 编写目的
本文档用于定义竞赛管理系统的业务范围、角色划分、功能需求、非功能需求、页面要求与数据处理原则，为产品设计、技术设计、开发与测试提供依据。

## 2. 业务目标
系统面向竞赛组织教师、管理人员与相关学生用户，建设覆盖赛前、赛中与赛后阶段的统一管理平台，实现信息统一、流程清晰、状态可追溯、资料可归档、统计可输出。

## 3. 用户角色
| 角色 | 说明 |
|---|---|
| 系统管理员 | 负责系统参数、权限、字典、账号管理 |
| 竞赛管理员/教师 | 负责赛事配置、报名审核、状态调整、通知、导出与归档 |
| 学生用户 | 负责报名、信息维护、材料提交、结果查看 |
| 审核人员 | 负责资格审核、材料复核、处理意见记录 |

## 4. 业务范围
### 4.1 赛前
- 赛事基础信息建立
- 学生信息导入与维护
- 报名开放、报名提交、材料提交
- 资格审核与补正
- 状态提醒与名单确认

### 4.2 赛中
- 名额管理
- 退赛、换人、补录
- 分组、通知与信息查询
- 特殊情况记录

### 4.3 赛后
- 成绩导入与核验
- 证书上传与归档
- 统计报表生成
- 教务材料整理与下载

## 5. 功能需求
### 5.1 基础资料管理
#### 5.1.1 学生信息管理
- 支持单条新增、批量导入、编辑、停用
- 字段包括学号、姓名、学院、专业、班级、联系方式、历史参赛记录等
- 支持按学号、姓名、专业、赛事参与情况查询

#### 5.1.2 赛事信息管理
- 支持赛事新增、编辑、发布、停用
- 支持维护赛事级别、主办单位、报名时间、比赛时间、材料要求、名额限制、规则附件等
- 支持赛事状态管理：草稿、待发布、报名中、审核中、已截止、已归档

### 5.2 报名管理
#### 5.2.1 在线报名
- 学生通过平台完成报名
- 支持表单校验与必填约束
- 支持根据学生历史信息自动回填基础字段
- 支持报名记录提交后查看当前状态

#### 5.2.2 批量导入导出
- 支持 Excel 模板导入报名数据
- 支持导出报名名单、审核结果、待补材料清单
- 导入时提供格式校验、错误提示、重复数据识别

### 5.3 材料提交与审核
- 支持上传身份证明、照片、证明材料等附件
- 支持审核通过、退回补正、驳回等处理结果
- 支持记录审核人、审核时间、处理意见
- 支持对未提交或补正未完成的记录生成待办提醒

### 5.4 资格与状态流转管理
- 支持退赛、换人、补录、替补转正等业务动作
- 各状态变更需记录操作人、时间、原因、前后状态
- 支持按赛事维度查看当前有效名单和历史流转记录

### 5.5 通知与消息中心
- 平台内提供通知公告、站内消息、待办提醒
- 支持面向赛事、角色、状态条件进行定向通知
- 支持消息已读状态记录
- 支持管理员查看消息发送记录与失败记录

### 5.6 赛后管理
- 支持成绩名单导入与匹配
- 支持证书上传、归档、检索与下载
- 支持生成获奖统计、参与统计、未获奖名单等报表
- 支持导出归档材料目录与提交记录

### 5.7 系统管理
- 用户、角色、菜单、权限管理
- 参数管理、字典管理
- 操作日志、登录日志
- 附件存储策略与数据备份配置

## 6. 非功能需求
### 6.1 易用性
- 页面结构清晰，信息层级明确
- 表单、表格、筛选、弹窗风格统一
- 常用操作路径简洁，减少多层跳转
- 错误提示、空状态提示、操作反馈可理解

### 6.2 界面要求
- 管理端页面遵循整洁、美观、大方的设计要求
- 保持 Ruoyi-Vue 原生风格基础上的统一布局与间距规范
- 表格页优先采用“查询区 + 工具栏 + 数据表格 + 分页”结构
- 详情页优先采用“摘要信息 + 标签页/分区块”结构
- 避免密集堆叠组件与无序颜色使用

### 6.3 性能要求
- 常规分页查询在合理数据量下应保持较顺畅响应
- 导入导出任务应具备进度提示或结果反馈
- 附件上传应支持基础大小限制与失败提示

### 6.4 安全与审计
- 基于角色的访问控制
- 关键操作留痕
- 登录与接口访问日志记录
- 导入、导出、状态调整等动作应可追溯

### 6.5 可维护性
- 模块边界明确
- 接口命名规范
- 数据字典统一维护
- 重要配置参数可通过环境变量或参数表管理

## 7. 业务闭环要求
本项目按完整细粒度流程进行建设，闭环范围至少包括：
1. 赛事创建
2. 报名发布
3. 学生报名
4. 材料提交
5. 材料审核与补正
6. 资格确认
7. 名额与状态调整
8. 平台通知与待办
9. 成绩与证书归档
10. 统计导出与历史查询

## 8. 页面清单建议
| 一级模块 | 二级页面 |
|---|---|
| 首页/工作台 | 待办概览、赛事进度、数据摘要 |
| 学生管理 | 学生列表、学生详情、导入记录 |
| 赛事管理 | 赛事列表、赛事配置、赛事详情 |
| 报名管理 | 报名列表、报名详情、导入导出 |
| 材料审核 | 材料列表、审核详情、补正记录 |
| 资格管理 | 状态调整、变更记录、有效名单 |
| 通知中心 | 公告管理、消息记录、待办规则 |
| 赛后管理 | 成绩管理、证书管理、归档资料 |
| 统计报表 | 参与统计、获奖统计、导出中心 |
| 系统管理 | 用户、角色、菜单、日志、参数 |

## 9. 约束条件
- 后端必须采用 Flask
- 前端采用 Ruoyi-Vue 原生管理端结构进行二次开发
- 主要运行环境为 Windows
- 部署流程应简洁，适合单机或小规模服务器环境
- 不采用外部 RPA 作为主链路能力

## 10. 验收导向
验收以“完整流程可运行、核心数据可追溯、关键页面可操作、导入导出可使用、部署可复现”为基本方向，不以单点演示功能替代系统整体能力。
